# V0.1 Development Plan

## Goal

Build a local-first Chrome/Firefox bookmark AI organizer for one user, with Google Chrome as the V0.1 write target.

The app must import Chrome and Firefox bookmarks, generate a cleanup plan, let the user review AI/rule suggestions, store accepted results locally, create restorable backups, and write accepted results into Google Chrome.

## Non-Negotiables

- Do not modify Chrome or Firefox original bookmark files.
- Always read from copied source files.
- Create backups before import, cleanup apply, Chrome write-back, and restore.
- V0.1 may write to Chrome bookmark bar only after backup, preview, and explicit confirmation.
- If Chrome is running during write-back, close it automatically and reopen it after writing.
- V0.1 must not write to Firefox.
- Do not require a server for V0.1.
- Do not require account login.
- Every AI change must be reviewable.
- Chrome bookmark-bar write-back is the primary apply path in V0.1.
- HTML export remains a fallback and portable backup.

## Phase 1: Project Foundation

- Initialize Tauri 2 app.
- Add React 18 + TypeScript.
- Add Tailwind CSS and shadcn/ui.
- Add local SQLite access.
- Add app config storage for:
  - selected browser and profile
  - AI provider
  - API key
  - export directory
  - backup directory

Done when:

- The desktop app launches.
- The UI has sidebar, main list area, and settings page.
- The app can create/open its local database.

## Phase 2: Chrome Import

- Detect Chrome profile root on Windows:
  - `%LOCALAPPDATA%\Google\Chrome\User Data`
- List profiles:
  - `Default`
  - `Profile 1`
  - `Profile 2`
  - other profile directories with a `Bookmarks` file
- Copy selected profile's `Bookmarks` JSON into app backup/import storage.
- Parse copied JSON.
- Extract:
  - bookmarks
  - folders
  - folder hierarchy
  - URLs
  - titles
  - date added
- Save an import batch.

Done when:

- The user's Chrome bookmarks appear in browser-switch.
- Original folder paths are preserved.
- Chrome can be used as the first low-risk test source.

## Phase 3: Firefox Import

- Detect Firefox profile root on Windows:
  - `%APPDATA%\Mozilla\Firefox\Profiles`
- List profiles with visible names/paths, skipping empty `places.sqlite` files.
- Prefer the default profile from `%APPDATA%\Mozilla\Firefox\profiles.ini`.
- Copy selected profile's `places.sqlite` into app backup/import storage.
- Read copied database.
- Extract:
  - bookmarks
  - folders
  - folder hierarchy
  - URLs
  - titles
  - created/modified timestamps
  - visit counts when available
- Save an import batch.

Done when:

- The user's Firefox bookmarks appear in browser-switch.
- Original folder paths are preserved.
- Re-import creates a new batch or updates existing records predictably.

## Phase 4: Backup And Restore

- Create source snapshot before every import.
- Create local app-state backup before cleanup apply.
- Create local app-state backup before restore.
- Add backup history table.
- Restore browser-switch local database from selected backup.

Done when:

- The user can see import and app-state backups.
- The user can restore browser-switch to a previous local state.
- The user can restore Chrome `Bookmarks` from a Chrome backup.
- Restore never modifies Firefox files.

## Phase 5: Quick Cleanup

- Implement URL normalization.
- Implement exact duplicate detection.
- Implement rule categories.
- Implement dead-link checking:
  - concurrency limit
  - timeout
  - retry once
  - HEAD then GET fallback
- Show cleanup summary.

Done when:

- The app can identify duplicates without AI.
- The app can mark likely dead links.
- The UI can filter by duplicates and dead links.

## Phase 6: AI Cleanup

- Add AI settings:
  - provider
  - base URL
  - model
  - API key
- Support OpenAI-compatible chat completion first.
- Build batch prompts for bookmarks.
- Validate and repair JSON responses.
- Cache AI outputs by content hash.
- Save proposed:
  - title
  - category
  - tags
  - summary
  - confidence
  - reason

Done when:

- The user can run AI cleanup on selected bookmarks or an import batch.
- AI results appear as proposals, not final changes.
- Failed AI items are recoverable/retryable.

## Phase 7: Review Queue

- Add review table/list.
- Show original vs proposed fields.
- Add actions:
  - accept selected
  - reject selected
  - edit selected
  - accept all visible
- Persist review status.

Done when:

- The user can approve only the changes they want.
- Accepted changes are stored separately from original imported data.

## Phase 8: Chrome Write-Back

- Let the user choose the target Chrome profile.
- Detect whether Chrome appears to be running.
- Create a backup of the target Chrome `Bookmarks` file.
- Generate write preview:
  - target profile
  - bookmark bar category folders
  - bookmark count
  - folder count
  - skipped duplicates
  - skipped/review-required items
- Write accepted bookmarks directly into Chrome bookmark bar.
- Do not create an extra `browser-switch` parent folder.
- Clear the current Chrome bookmark bar before writing.
- Reopen Chrome after writing.
- Support restoring the previous Chrome backup.

Done when:

- Chrome cleanup results can be written to Chrome bookmark bar after confirmation.
- Firefox cleanup results can be written to Chrome bookmark bar after confirmation.
- No extra `browser-switch` parent folder is created.
- Chrome backups can be restored.

## Phase 9: Export

- Generate Netscape bookmark HTML.
- Export accepted title/category structure.
- Keep original URL.
- Put rejected/unreviewed items in their original folders or a configurable fallback folder.

Done when:

- Chrome and Firefox can import the exported HTML.
- Exported file contains cleaned structure and titles.
- Firefox profile is still untouched.
- Current Chrome bookmark bar is cleared before write.

## Recommended First Build Order

1. Scaffold desktop app.
2. Implement Chrome profile detection.
3. Import Chrome `Bookmarks` JSON.
4. Render bookmark list.
5. Add backup history and local restore.
6. Implement Firefox profile detection.
7. Import copied `places.sqlite`.
8. Add duplicate detection.
9. Add AI settings and one AI cleanup batch.
10. Add review queue.
11. Add Chrome bookmark-bar write-back.
12. Add Chrome backup restore.
13. Add HTML export fallback.

# V0.1 Development Plan

## Goal

Build a local-first Firefox bookmark AI organizer for one user.

The app must import Firefox bookmarks, generate a cleanup plan, let the user review AI/rule suggestions, store accepted results locally, and export cleaned bookmarks as HTML.

## Non-Negotiables

- Do not modify Firefox's original `places.sqlite`.
- Do not require a server for V0.1.
- Do not require account login.
- Every AI change must be reviewable.
- Export is the only way to get cleaned bookmarks back into Firefox in V0.1.

## Phase 1: Project Foundation

- Initialize Tauri 2 app.
- Add React 18 + TypeScript.
- Add Tailwind CSS and shadcn/ui.
- Add local SQLite access.
- Add app config storage for:
  - selected Firefox profile
  - AI provider
  - API key
  - export directory

Done when:

- The desktop app launches.
- The UI has sidebar, main list area, and settings page.
- The app can create/open its local database.

## Phase 2: Firefox Import

- Detect Firefox profile root on Windows:
  - `%APPDATA%\Mozilla\Firefox\Profiles`
- List profiles with visible names/paths.
- Copy selected profile's `places.sqlite` into app temp storage.
- Read copied database.
- Extract:
  - bookmarks
  - folders
  - folder hierarchy
  - URLs
  - titles
  - created/modified timestamps
  - visit counts when available
- Save an import batch.

Done when:

- The user's Firefox bookmarks appear in browser-switch.
- Original folder paths are preserved.
- Re-import creates a new batch or updates existing records predictably.

## Phase 3: Quick Cleanup

- Implement URL normalization.
- Implement exact duplicate detection.
- Implement rule categories.
- Implement dead-link checking:
  - concurrency limit
  - timeout
  - retry once
  - HEAD then GET fallback
- Show cleanup summary.

Done when:

- The app can identify duplicates without AI.
- The app can mark likely dead links.
- The UI can filter by duplicates and dead links.

## Phase 4: AI Cleanup

- Add AI settings:
  - provider
  - base URL
  - model
  - API key
- Support OpenAI-compatible chat completion first.
- Build batch prompts for bookmarks.
- Validate and repair JSON responses.
- Cache AI outputs by content hash.
- Save proposed:
  - title
  - category
  - tags
  - summary
  - confidence
  - reason

Done when:

- The user can run AI cleanup on selected bookmarks or an import batch.
- AI results appear as proposals, not final changes.
- Failed AI items are recoverable/retryable.

## Phase 5: Review Queue

- Add review table/list.
- Show original vs proposed fields.
- Add actions:
  - accept selected
  - reject selected
  - edit selected
  - accept all visible
- Persist review status.

Done when:

- The user can approve only the changes they want.
- Accepted changes are stored separately from original imported data.

## Phase 6: Export

- Generate Netscape bookmark HTML.
- Export accepted title/category structure.
- Keep original URL.
- Put rejected/unreviewed items in their original folders or a configurable fallback folder.

Done when:

- Firefox can import the exported HTML.
- Exported file contains cleaned structure and titles.
- Original Firefox profile is still untouched.

## Recommended First Build Order

1. Scaffold desktop app.
2. Implement Firefox profile detection.
3. Import `places.sqlite`.
4. Render bookmark list.
5. Add duplicate detection.
6. Add AI settings and one AI cleanup batch.
7. Add review queue.
8. Add HTML export.
